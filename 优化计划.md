3. **Filter 与 Controller 响应格式不一致**：Filter 用 `sendError`，Controller 用 `Result` JSON。  
4. **issuer 仍为旧值** `fangzhengyuan` → 待清理（见 §1.1）。

### 2.5 双令牌 + Redis 会话（推荐，与 D4 一起做）

> **目的**：仅靠 JWT `exp` 无法「登出即失效」或「改密后踢掉旧会话」；接入 Redis 后顺手做双令牌，面试可讲「无状态 JWT + 有状态会话吊销」。

**建议分工**：

| 令牌 | 形态 | 有效期 | 存储 |
|------|------|--------|------|
| Access Token | JWT（含 `jti`、`sub`） | 15~30 分钟 | 仅客户端 |
| Refresh Token | 随机 UUID 或 JWT | 7 天 | Redis `mes:auth:refresh:{userId}:{jti}` |

**Filter 校验顺序**（每次请求）：

1. 验 JWT 签名 + `exp`  
2. 查 Redis：`mes:auth:access:{jti}` 存在（或不在黑名单 `mes:auth:blacklist:{jti}`）  
3. 从 `mes:user:auth:{username}` 取 `LoginUser`（含角色）

**接口**：

- `POST /api/auth/login` → `{ accessToken, refreshToken, expiresIn }`  
- `POST /api/auth/refresh` → 校验 refresh 在 Redis 且未吊销 → 轮换 refresh + 新发 access  
- `POST /api/auth/logout` → 删 refresh；可选将当前 access `jti` 加入黑名单至过期  

**实施顺序**：D4 Redis 配置通 → 2.2 LoginUser 缓存 → 2.5 双令牌（不要先于 Redis 硬写）。
